# AX.25

AX.25 is the link-layer protocol packet radio rides on.  LinBPQ
speaks both v2.0 and v2.2 (default v2.2; force v2.0 with the
top-level `OnlyVer2point0=1` cfg keyword).  This page covers what
LinBPQ does with AX.25 in practice — frame format, the timers
exposed in `bpq32.cfg`, and how to think about the L2 knobs.

For the wire-format spec itself the canonical references are at
[packethacking/ax25spec][spec] (TAPR AX.25 v2.2 plus the various
KISS extensions); LinBPQ implements that spec.

[spec]: https://github.com/packethacking/ax25spec

## Frame layout (recap)

```
| Flag | Address (14–70 bytes) | Control | PID | Info | FCS | Flag |
```

- **Flag** — `0x7E` start/end.
- **Address** — 7 bytes per call (6 chars padded with spaces +
  SSID byte).  Always two calls (dest, source); 0–8 digipeater
  calls follow.  The last byte's bit 0 is the end-of-address marker.
- **Control** — frame type (I, RR, RNR, REJ, SREJ, U-frame
  variants).
- **PID** — protocol ID for I/UI frames: `0xCC` IP, `0xCF`
  NET/ROM, `0xF0` text, etc.
- **Info** — payload.
- **FCS** — CRC-16-CCITT.

## Per-port L2 timers (in `bpq32.cfg`)

These are the `PORT`-block knobs that map directly onto AX.25
timer values (or KISS-TNC parameters when the L2 is offloaded):

| Cfg keyword | AX.25 name | Meaning |
|---|---|---|
| `MAXFRAME=n` | `MAXFRAME` (window) | Max outstanding I-frames before waiting for ACK.  v2.0: 1–7.  v2.2: 1–127. |
| `FRACK=ms` | `T1` | Frame-acknowledge timer.  How long we wait for the peer to ACK an I-frame before retransmit. |
| `RESPTIME=ms` | `T2` | Delayed-ACK timer.  How long we hold an ACK back hoping to piggyback. |
| `RETRIES=n` | `N2` | Retry limit.  After this many `FRACK` timeouts the link drops. |
| `T3=s` (top-level) | `T3` | Link-validation timer.  Idle longer than this and we send `RR(P)` to check the peer. |
| `IDLETIME=s` | — | LinBPQ-specific.  Idle longer and the link disconnects. |
| `PACLEN=n` | `PACLEN` | Max info-field bytes per I-frame.  236 is the practical ceiling for NET/ROM. |
| `TXDELAY=ms` | KISS `TXDELAY` | Transmit-key delay before the first frame goes out. |
| `TXTAIL=ms` | KISS `TXTAIL` | Transmit-key hold after the last frame. |
| `SLOTTIME=ms` | KISS `SLOTTIME` | CSMA slot interval. |
| `PERSIST=n` | KISS `PERSIST` | Probability-to-transmit (0–255). |
| `FULLDUP=0/1` | KISS `FULLDUP` | Duplex flag passed to TNC.  HDLC: TX-during-DCD. |

`TXDELAY` / `TXTAIL` are entered in milliseconds in `bpq32.cfg`
but written to KISS TNCs in 10 ms units (LinBPQ divides by 10
before sending).

## Tuning rough rules

- **`FRACK`** = round-trip + a little.  On 1200 baud VHF with no
  digis: 3000–5000 ms.  Through one digi: 5000–7000 ms.  HF
  Pactor: tens of seconds.
- **`MAXFRAME`** higher = more throughput, more retransmit cost
  on a lossy link.  Start at 4 on VHF, 1–2 on HF.
- **`RETRIES`** × `FRACK` = max time before a stuck link drops.
  10 × 5000 ms = 50 seconds is reasonable; tune higher if your
  channel is bursty.
- **`T3`** wants to be longer than the longest plausible quiet
  period that's still a live link.  Default 180 s is fine for
  routine use.

## Per-port behaviour knobs

| Keyword | Effect |
|---|---|
| `BBSFLAG=NOBBS` | Refuse direct BBS connects on this port. |
| `L3ONLY=1` | Refuse user-initiated downlink connects (HOST sessions still allowed). |
| `INP3ONLY=1` | Don't accept NODES broadcasts on this port — INP3 routing only. |
| `IGNOREUNLOCKEDROUTES=1` | Only locked ROUTES entries can broadcast NODES on this port. |
| `UIONLY=1` | No connected-mode at all on this port — UI only (APRS digi etc.). |
| `MAXDIGIS=n` | Cap on digis in the path of an inbound connect. |
| `VALIDCALLS=...` | Allowlist by callsign. |
| `USERS=n` | Cap on concurrent L2 sessions on this port. |
| `INTERLOCK=n` | Group ports so only one in the group can TX at a time (Pactor + WINMOR sharing one radio). |
| `TXPORT=n` | Send TX traffic out on a different port (single-TX, multi-RX setups). |

## Compression (optional)

LinBPQ supports an L2 compression scheme (separate from the L4
compression used over NET/ROM):

```ini
L2Compress=1
L2CompMaxframe=3
L2CompPaclen=236
```

L2 compression only helps over noisy / slow links; on healthy
9k6 packet it tends to be a wash.
